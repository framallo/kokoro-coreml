# Fixed-Cost Latency Fit

Collected: 2026-06-06.

This note records the warmed frontier diagnostic generated by
`scripts/external_bakeoff/summarize_fixed_cost_latency_fit.py`. The model is:

`latency_ms = fixed_ms + slope_ms_per_audio_s * canonical_audio_duration_s`

Per-bucket warmed rows remain the benchmark source of truth. This fit is only a
way to separate short-bucket fixed/boundary effects from duration-scaled
generator cost.

## Decision

- MLX is not the current warmed Mac blocker. Corrected warmed Config F is faster
  on every full-duration MLX bucket, and MLX has no valid full-duration `3s`
  Mac row.
- laishere remains the lower-end strict blocker. Config F loses short buckets on
  M1/M2 Air but scales better with duration, which is consistent with the
  current boundary-collapse thesis.
- Negative fixed terms and low `R^2` rows mean the global linear fit is
  heuristic. Do not promote a benchmark claim from the fit alone.
- After the overlap + HAR-post rewrite projection, another `1-3%` host tweak is
  below the remaining Irvine `3s/7s` budget. New strict candidates should remove
  a Core ML call boundary, repair the HAR/STFT source contract for a
  single-package source/body path, or materially reduce generator-stage cost.
